<h3>Example notes</h3>

This example illustrates a Task used to request patient consent for fulfillment of a specialty product

Key elements are...

- **Task.status**: *requested*
- **Task.code**: *obtain-consent*
  - Characterizes the requested user action (and directs the EHR to make the consent form referenced in Task.input available to the user)
- **Task.description**: *Have the patient complete the attached consent form and capture a PDF of the signed form in a Consent*
  - User-directed text describing the task to be performed
- **Task.for**: Reference to the patient resource that resides in the Data Source system
- **Task.requester**:  The organization submitting the Task
- **Task.owner**: The prescriber
- **Task.reasonReference**: The prescription to which the task pertains
- **Task.input**: A reference to a Consent resource containing the consent form that is appropriate for the prescribed product
  - **Task.input.type**: *consent-form-reference*
  - **Task.input.valueReference**: *specialty-rx-consent-requested-1*
